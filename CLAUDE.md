# Reglas de despliegue

- Rama de trabajo diaria: `dev`
- NUNCA hacer `git push origin main` directamente
- Para guardar avances (gratis, ilimitado): `git save`
- Para publicar en vivo (gasta créditos de Netlify): `git publish`
- Solo usar `git publish` cuando JR confirme que quiere que el cambio sea público
